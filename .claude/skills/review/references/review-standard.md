# Review Standards

Review the change, not the person; be specific and suggest fixes.

## Check
- [ ] Does what it claims; edge cases & failure paths
- [ ] Tests meaningful, not just present
- [ ] Naming & readability
- [ ] Security & data handling
- [ ] Performance on hot paths
- [ ] Docs updated if needed

## Labels
- **Blocking**: must fix before merge.
- **Suggestion**: recommended, non-blocking improvement.
- **Nit**: optional polish.
- **Question**: needs clarification.
